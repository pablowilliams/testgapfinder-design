Ranked test debt

Files are ranked by a test-debt score that combines uncovered-line count, longest uncovered span, and a path-based severity hint (billing, payment, auth paths rank higher than utils, logging). This demo uses the coverage file only. The full pipeline (see design report (PDF, 56 KB)) also considers git churn, cyclomatic complexity, and call-graph centrality.

    Heuristic descriptions

    These descriptions are generated from line ranges in your coverage file. They are templates, not an analysis of your code — the LLM reasoner described in the design report is not in this demo.

    TestGapFinder is an open-source design project. Analysis runs client-side against the standard Cobertura schema. Coverage data never leaves your browser. View source on GitHub.
